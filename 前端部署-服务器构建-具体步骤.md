# 前端部署（服务器构建）— 具体步骤

在 107 上拉代码并构建，本机只做 git push，避免慢速上传 dist。

---

## 一、一次性准备（在 107 上做一次）

### 1. 登录跳板机

在本机 PowerShell 执行：

```powershell
ssh root@107.174.174.123
```

输入密码后，进入 107 的终端。

---

### 2. 检查是否已安装 Node（在 107 上执行）

```bash
node -v
npm -v
```

若提示找不到命令，先安装：

```bash
apt update
apt install -y nodejs npm
```

再执行一次 `node -v`、`npm -v` 确认。

---

### 3. 克隆仓库并安装依赖（在 107 上执行）

```bash
cd /tmp
rm -rf younov-build
git clone --depth 1 https://github.com/fc5207646-stack/YouNov.git younov-build
cd younov-build/work/frontend
npm install
```

若仓库为私有，会提示输入 GitHub 用户名和 token。  
等待 `npm install` 完成（只需做一次）。

---

### 4. 确认网站目录存在（在 107 上执行）

```bash
mkdir -p /var/www/younov/dist
```

若该目录已存在可跳过。  
准备阶段到此结束，可输入 `exit` 退出 SSH。

---

## 二、以后每次更新前端

### 步骤 1：本机 — 改代码并推送

在 **本机 PowerShell** 执行（把改过的文件都 add 进去）：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov"
git add work/frontend
git status
git commit -m "frontend: update"
git push origin master
```

若没有改动，`git status` 会提示无变更，可不用 commit/push。

---

### 步骤 2：107 上 — 拉代码、构建、发布

**2.1** 登录 107：

```powershell
ssh root@107.174.174.123
```

**2.2** 在 107 上依次执行：

```bash
cd /tmp/younov-build
git pull
cd work/frontend
npm run build
cp -r dist/* /var/www/younov/dist/
```

**2.3** （可选）若站点要求特定权限：

```bash
chown -R www-data:www-data /var/www/younov/dist
```

**2.4** 验证：浏览器打开 https://younov.com ，按 Ctrl+Shift+R 强制刷新。

---

## 三、命令汇总（复制用）

**本机（每次改完前端后）：**

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov"
git add work/frontend
git commit -m "frontend: update"
git push origin master
```

**107 上（每次部署时）：**

```bash
cd /tmp/younov-build && git pull && cd work/frontend && npm run build && cp -r dist/* /var/www/younov/dist/
```

---

## 四、若 107 上 younov-build 被删掉

重新做「一、一次性准备」里的 **1～3** 步即可（重新 clone、`cd work/frontend`、`npm install`），然后再按「二」操作。

若网站根目录不是 `/var/www/younov/dist`，把上面所有该路径改成你实际使用的路径。
