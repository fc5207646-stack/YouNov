# 小说数据来源说明

## 前端数据从哪来

首页 **Featured**、**Trending** 和 **Library/Free** 列表都来自**后端 API**，前端没有写死任何小说：

- 首页：请求 `/api/leaderboard`（mostRead + topRated），失败时用 `/api/novels?take=18&skip=0` 兜底。
- Library / Free：请求 `/api/novels`、`/api/free-novels`（带分类、分页等参数）。

所以你看到的“测试小说”（包括英文书名等）都是**当前请求到的那台 API 所连数据库**里的数据。

---

## 为什么只有“测试小说”、没有你的 38 本

常见情况有两种：

### 情况 1：请求打到了“测试库”所在的 API

- 你有两台 API（10.50.0.2、10.50.0.3），可能一台连的是**有 38 本小说的库**，另一台连的是**只有 seed 测试数据的库**。
- 网站通过 107 的 Nginx 把 `/api` 反代到其中一台（或轮询）。若反代到的是“测试库”那台，前端就只看到测试小说。

**排查：**

在 107 上执行（或在本机用浏览器访问）：

```bash
curl -s https://younov.com/api/leaderboard | head -500
curl -s https://younov.com/api/novels?take=5
```

看返回的 `mostRead`/`topRated`/`items` 里是 38 本里的书名，还是只有测试/英文书。再在**两台 API 机上**分别执行：

```bash
# 在 10.50.0.2 上
curl -s http://127.0.0.1:8080/api/novels?take=3
# 在 10.50.0.3 上
curl -s http://127.0.0.1:8080/api/novels?take=3
```

对比哪台返回的是 38 本、哪台是测试数据，就能确定“有真实数据的是哪台”。

**处理：**

- 让 Nginx 的 `proxy_pass` **只指向“有 38 本”的那台 API**（或先只保留这一台），这样前端就只会拿到真实小说数据。
- 或者把“测试库”那台也改成连 38 本所在的数据库（改 `.env` 里 `DATABASE_URL`，重启 API），再保持双机。

### 情况 2：当前库就是测试库，38 本在别处

- 若 38 本在另一个库/另一台机，当前 API 连的库只有 seed 的测试小说，那前端自然只有测试数据。

**处理：**

- 把 38 本导入到当前 API 使用的数据库（用 Admin 后台批量创建/导入，或从原库导出再导入），或  
- 把当前 API 的 `DATABASE_URL` 改成指向“有 38 本”的那个库，重启 API。

## 如何增加/管理“真实小说”数据

1. **用 Admin 后台**  
   登录管理后台（如 `/admin/login`），在「小说管理」里新增、编辑小说（封面、简介、分类、章节等）。这些会写入当前 API 所连的数据库。

2. **用 seed / 脚本**  
   在 `work/api/prisma/seed.js` 里加你的小说（或写脚本用 Prisma 插入），然后在**连了目标库的那台 API 机**上执行：  
   `npm run seed` 或 `node scripts/your-import.js`。  
   注意：seed 当前只有一条测试小说和 admin 账号，若你要 38 本，建议用脚本或 Admin 批量导入。

3. **双 API 回退**  
   若一台 API 有 38 本、一台是测试库，且已按仓库里的双 API 回退逻辑配置了 `PEER_API_URL`，则本机没数据时会请求对端；但要**稳定**显示 38 本，最好让前端请求到的“主”API 就是连 38 本的那台（见上面 Nginx / DATABASE_URL 配置）。

---

## 小结

| 现象           | 可能原因                     | 做法 |
|----------------|------------------------------|------|
| 前端只有测试小说 | 请求到了连“测试库”的 API     | 确认 Nginx 反代到哪台；让请求只到“有 38 本”的那台，或把两台都连到 38 本所在库 |
| 前端只有测试小说 | 当前 API 连的库里只有 seed 数据 | 用 Admin 或脚本把 38 本导入当前库，或把 DATABASE_URL 改为 38 本所在库 |

前端没有“测试小说”假数据，只要接口返回的是 38 本所在库的数据，首页和 Library/Free 就会显示真实小说资源。
