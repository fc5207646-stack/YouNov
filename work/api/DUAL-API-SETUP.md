# 双 API / 双库回退说明

## 行为

- **GET /api/leaderboard**、**GET /api/novels**、**GET /api/free-novels** 会先查本机数据库。
- 若本机返回**有数据**（列表非空），直接返回给前端。
- 若本机返回**空**，且配置了 `PEER_API_URL`，则向对端请求**同路径+同 query**，对端带上 `X-Internal-Fallback: 1` 后**不再回退**，避免成环。
- 若对端返回有数据，则把对端结果返回给前端；否则返回本机空结果。

这样两台 API/两台库中，只要有一台有数据，前端就能拿到有数据的那一侧结果。

## 配置

在每台 API 服务器的 `.env` 中设置对端地址（不要带尾部斜杠）：

- **服务器 A**（例如 10.50.0.2）：`PEER_API_URL=http://10.50.0.3:8080`
- **服务器 B**（例如 10.50.0.3）：`PEER_API_URL=http://10.50.0.2:8080`

两台之间需能通过上述地址访问（内网互通）。不配置 `PEER_API_URL` 则不做回退，仅返回本机数据。

## 部署后

修改 `.env` 后重启 API 服务即可，例如：

```bash
systemctl restart younov-api
```
