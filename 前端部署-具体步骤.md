# 前端部署 — 具体步骤（解决白屏）

把本机构建好的前端（含 IpChecker、AuthProvider 修复）上传到服务器，使 https://younov.com 正常显示。

---

## 步骤一：本机构建

在 **PowerShell** 中执行：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov\work\frontend"
npm run build
```

若项目有 `build:clean`（先清再构建），也可用：

```powershell
npm run build:clean
```

构建成功后，会在 `work\frontend\dist` 下生成 `index.html` 和 `assets` 文件夹（内有若干 .js、.css 等）。

---

## 步骤二：上传到服务器

服务器信息（与你当前环境一致）：

- **跳板机**：107.174.174.123（root）
- **网站根目录**：/var/www/younov/dist

任选一种方式即可。

### 方式 A：用脚本上传（会提示输入密码）

在项目根目录执行（脚本会先构建再上传前端到 107）：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov"
.\deploy-categories-update.ps1
```

若未配置 SSH 密钥，会多次提示 `root@107.174.174.123's password:`，在**当前窗口**输入跳板机 root 密码并回车。  
脚本会执行：本机构建 → 上传 `dist/index.html` 和 `dist/assets/*` 到 107 的 `/var/www/younov/dist/` → 清理旧 JS。

若你**只打算更新前端**、不想动 API，可手动执行下面两条（在已完成步骤一的前提下）：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov"
$dist = ".\work\frontend\dist"
scp -o StrictHostKeyChecking=accept-new -i "$env:USERPROFILE\.ssh\id_ed25519" "$dist\index.html" root@107.174.174.123:/var/www/younov/dist/
scp -o StrictHostKeyChecking=accept-new -i "$env:USERPROFILE\.ssh\id_ed25519" -r "$dist\assets\*" root@107.174.174.123:/var/www/younov/dist/assets/
```

出现密码提示时，在同一窗口输入密码并回车。  
上传时可在 `scp` 后加 `-o Compression=yes` 减少传输量，例如：

```powershell
scp -o StrictHostKeyChecking=accept-new -o Compression=yes -i "$env:USERPROFILE\.ssh\id_ed25519" ...
```

### 方式 B：用 WinSCP 上传（图形界面）

1. 打开 **WinSCP**，新建站点：
   - 协议：SFTP
   - 主机：107.174.174.123
   - 用户名：root
   - 密码：跳板机 root 密码
2. 登录后，在右侧进入 **/var/www/younov/dist**。
3. 左侧进入本机目录：`c:\Users\Administrator\Desktop\网站源码\YouNov\work\frontend\dist`。
4. 把左侧的 **index.html** 拖到右侧，覆盖。
5. 把左侧 **assets** 文件夹里的**全部文件**拖到右侧 **assets** 里，覆盖（或先删掉右侧 assets 下旧文件再拖入整份 assets）。

---

## 步骤三：确认

1. 浏览器访问：https://younov.com  
2. 若仍白屏：按 **Ctrl+Shift+R** 强制刷新，或清除站点数据后再打开。  
3. 打开 F12 → Network，筛选「全部」，确认 **assets/*.js** 为 200，无 404。

---

## 小结

| 步骤 | 操作 |
|------|------|
| 一 | 本机 `cd work\frontend` → `npm run build`（或 `build:clean`） |
| 二 | 任选：A 运行 `.\deploy-categories-update.ps1` 或手动 scp；B 用 WinSCP 把 `dist` 内容上传到 107 的 `/var/www/younov/dist` |
| 三 | 访问 younov.com，必要时强制刷新，检查 Network 无 404 |

路径以你实际为准：若网站根目录不是 `/var/www/younov/dist`，把上面所有该路径改成你服务器上的路径。

## 提速：在服务器上构建（本机只 push，不传大文件）

若本机到 107 上传很慢，可改为**在 107 上拉代码并构建**，本机只做 `git push`（通常很快）。

### 一次性准备（在 107 上执行一次）

```bash
ssh root@107.174.174.123
# 在 107 上：
cd /tmp
git clone https://github.com/fc5207646-stack/YouNov.git younov-build
cd younov-build/work/frontend
npm install
# 若 107 上没有 node，需先安装：apt update && apt install -y nodejs npm
```

### 以后每次更新前端

**本机**（只推代码，几秒内完成）：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov"
git add work/frontend/src work/frontend/index.html work/frontend/vite.config.js work/frontend/package.json
git add work/frontend/public 2>$null; git add work/frontend/*.js 2>$null
git commit -m "frontend: update"
git push origin master
```

**107 上**（拉代码、构建、发布）：

```bash
cd /tmp/younov-build
git pull
cd work/frontend
npm run build
cp -r dist/* /var/www/younov/dist/
# 若需修正权限：chown -R www-data:www-data /var/www/younov/dist
```

这样耗时主要在 107 本机的 `npm run build`，不再从你本机上传整份 dist，整体会快很多。
