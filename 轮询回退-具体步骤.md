# 轮询回退 — 具体步骤

按顺序做下面几步，即可实现：先判断数据在哪台，再返回给前端。

---

## 步骤 1：在两台 API 上配置对端地址（PEER_API_URL）

**1.1 登录 10.50.0.2**

从本机或 107 跳板机执行：

```bash
ssh root@107.174.174.123
ssh root@10.50.0.2
```

**1.2 在 10.50.0.2 上编辑 .env**

```bash
cd /opt/younov/api
nano .env
```

在文件里增加或修改一行（对端是 10.50.0.3）：

```
PEER_API_URL=http://10.50.0.3:8080
```

保存退出（nano：Ctrl+O 回车，Ctrl+X）。

**1.3 重启 10.50.0.2 的 API**

```bash
systemctl restart younov-api
systemctl status younov-api
```

确认状态为 `active (running)`。

**1.4 在 10.50.0.3 上做同样配置（对端是 10.50.0.2）**

```bash
# 从 107 登录 10.50.0.3
ssh root@10.50.0.3
cd /opt/younov/api
nano .env
```

增加或修改：

```
PEER_API_URL=http://10.50.0.2:8080
```

保存退出后：

```bash
systemctl restart younov-api
systemctl status younov-api
```

---

## 步骤 2：确认至少一台 API 连到“有 38 本”的数据库

**2.1 确定 38 本小说在哪台数据库**

确认你的两台数据库服务器的 IP/主机名，以及 38 本所在库的：主机、端口、库名、用户名、密码。

**2.2 在“有 38 本”的那台数据库上确认有数据**

若能直连该库，用客户端或命令行查一下小说表是否有约 38 条（或你期望的数量）。

**2.3 让至少一台 API 的 DATABASE_URL 指向该库**

例如 38 本在数据库服务器 `db1.example.com`，端口 5432，库名 `younov`，用户 `app`，密码 `xxx`，则在该 API 的 `.env` 里：

```
DATABASE_URL="postgresql://app:xxx@db1.example.com:5432/younov"
```

- 若 38 本在**第一台数据库**：让 **10.50.0.2** 的 `DATABASE_URL` 指向它（或 10.50.0.3 指向它，至少一台即可）。
- 若 38 本在**第二台数据库**：让其中一台 API 的 `DATABASE_URL` 指向第二台。

修改后重启该 API：

```bash
systemctl restart younov-api
```

---

## 步骤 3：在 107 上确认 Nginx 把 /api 反代到两台 API（轮询）

**3.1 登录 107**

```bash
ssh root@107.174.174.123
```

**3.2 查看 younov 配置**

```bash
cat /etc/nginx/sites-enabled/younov
```

**3.3 确保有 upstream 且 location /api/ 已启用**

若没有或被注释，在 `server { ... }` 里、且在 `location /` 之前添加或取消注释：

```nginx
upstream younov_api {
    server 10.50.0.2:8080;
    server 10.50.0.3:8080;
    keepalive 32;
}

location /api/ {
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Connection "";
    proxy_pass http://younov_api;
}
```

若原配置里 upstream 名字是 `api_upstream`，则 `proxy_pass` 写 `http://api_upstream;`，与 upstream 名字一致即可。

**3.4 测试并重载 Nginx**

```bash
nginx -t
systemctl reload nginx
```

---

## 步骤 4：验证

**4.1 在 107 上测接口（应返回有数据的一侧）**

```bash
curl -s "https://younov.com/api/novels?take=5"
```

若配置正确且至少一台 API 连的库有 38 本，这里应看到 `items` 数组非空。

**4.2 多请求几次**

因 Nginx 轮询，有时会打到“无数据”的那台，该 API 会向对端要数据并返回，所以多试几次都应能拿到数据（或至少有一次拿到）。

**4.3 浏览器访问**

打开 https://younov.com ，首页 Featured / Trending 和 Library 应能显示小说。
