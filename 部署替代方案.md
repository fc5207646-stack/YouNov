# 部署速度为 0 / 卡住 — 其他办法

## 现象说明

脚本执行后出现 `0% 0 0.0KB/s - stalled`，多半是 **SCP 在等密码**，但 PowerShell 里有时不显示 `password:` 提示。

---

## 办法一：在当前窗口直接输入密码（先试）

1. **不要关窗口**，看脚本是否停在那行（0% / stalled）。
2. **在当前 PowerShell 窗口里**直接输入跳板机 **root 的密码**（输入时不会显示任何字符，正常）。
3. 输完后按 **Enter**。
4. 若成功，会开始上传；之后可能再问 1～2 次密码，同样在该窗口输入并回车。

若这样能跑通，说明只是提示没显示，以后照此操作即可。若仍一直 0%，用下面其他办法。

---

## 办法二：配置 SSH 密钥（推荐，一劳永逸）

配置好后不再需要输入密码，也不会再卡在 0%。

**步骤概要：**

1. 本机生成密钥（若已有 `C:\Users\Administrator\.ssh\id_ed25519` 可跳过）：
   ```powershell
   ssh-keygen -t ed25519 -f "$env:USERPROFILE\.ssh\id_ed25519" -N '""'
   ```

2. 把公钥拷到跳板机（会提示输入**一次**密码）：
   ```powershell
   Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub" | ssh root@107.174.174.123 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
   ```

3. 验证免密：下面命令应直接输出 `ok` 且不再问密码：
   ```powershell
   ssh -i "$env:USERPROFILE\.ssh\id_ed25519" root@107.174.174.123 "echo ok"
   ```

4. 再执行 `.\deploy-api-quick.ps1`，应不再卡住、不再问密码。

更细步骤见：`deploy-api-quick-步骤.md` 里的「二、可选：配置 SSH 密钥」。

---

## 办法三：在服务器上拉代码（不依赖本机 SCP）

适合：本机到服务器网络不稳定，或你更习惯在服务器上操作。

**思路**：代码放到 Git（如 GitHub/Gitea），在跳板机或 API 机上 `git pull` 得到最新 `index.js`，再拷贝到两台 API 并重启。

1. 在跳板机 107.174.174.123 上 SSH 登录后执行（把仓库地址换成你的）：
   ```bash
   cd /tmp
   git clone --depth 1 https://你的仓库地址 younov-tmp
   cp younov-tmp/work/api/src/index.js /path/to/api/src/   # 或先 scp 到 10.50.0.2/10.50.0.3
   ```
   若 API 在 10.50.0.2/10.50.0.3 且跳板机可 SSH 到它们，可以在跳板机用 scp 把 `/tmp/younov-tmp/work/api/src/index.js` 拷到两台机的 `/opt/younov/api/src/index.js`，再分别 ssh 上去 `systemctl restart younov-api`。

2. 或在**每台 API 机**上若有克隆过的仓库，直接：
   ```bash
   cd /opt/younov/api  # 或你的仓库目录
   git pull
   systemctl restart younov-api
   ```

这样完全不依赖从本机 Windows 用 SCP 传文件，避免本机到跳板机之间的 0% 问题。

---

## 办法四：用图形工具上传，再 SSH 重启

1. 用 **WinSCP** 或 **FileZilla** 连到 107.174.174.123（或若能直连则连 10.50.0.2/10.50.0.3），在图形界面里输入密码上传 `work\api\src\index.js` 到目标机的 `/opt/younov/api/src/index.js`。
2. 用 **PuTTY** 或 PowerShell `ssh` 登录同一台机，执行：
   ```bash
   systemctl restart younov-api
   ```
3. 若 API 在两台机，对两台都做一次上传 + 重启。

这样密码在图形界面里输入，不会出现「看不到提示、速度为 0」的情况。

---

## 小结

| 办法     | 适用情况           | 效果           |
|----------|--------------------|----------------|
| 一、直接输密码 | 只是没看到密码提示 | 立刻可继续部署 |
| 二、SSH 密钥 | 想长期免密、避免卡住 | 推荐，一次配置长期有效 |
| 三、服务器拉代码 | 本机到服务器网络差 | 不依赖本机 SCP |
| 四、WinSCP 等 | 习惯图形界面       | 密码在窗口里输入，不会 0% 卡住 |

建议先试**办法一**；若仍为 0%，优先做**办法二**（SSH 密钥），再跑 `.\deploy-api-quick.ps1` 即可。
