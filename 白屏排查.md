# 白屏排查步骤

## 1. 先看浏览器控制台（必做）

1. 打开 https://younov.com ，按 **F12** 打开开发者工具。
2. 切到 **Console** 面板：看有没有**红色报错**（例如 Failed to load resource、Uncaught 等）。
3. 切到 **Network** 面板，**把筛选改成「All」或「全部」**（不要只选 XHR/Fetch），然后刷新页面：
   - 看是否有请求**红色**（状态码 404 或 failed）。重点看 **.js**、**.css** 是否 404。
   - 若 `index.html` 能打开，但下面某个 `assets/xxx.js` 是 404，说明前端资源路径不对或服务器上没有对应文件。
   - 若只看到名为 `num` 等少数请求，多半是筛选掉了文档和脚本；改成「全部」后再看一遍。

把看到的报错或 404 的 URL 记下来，便于排查。

---

## 2. 常见原因与处理

### A. 前端 JS/CSS 404（最常见）

**原因**：服务器上的前端是旧构建，或根本没部署过当前项目的 `dist`。

**处理**：在**本机**构建前端，再把 `dist` 上传到网站所在服务器（例如 107.174.174.123 的 `/var/www/younov/dist`）。

本机执行：

```powershell
cd "c:\Users\Administrator\Desktop\网站源码\YouNov\work\frontend"
npm run build
```

然后用 WinSCP / 其他方式把 `work\frontend\dist` 里的**全部文件**上传到服务器站点根目录（覆盖原有文件）。若你之前用脚本部署过前端，可再跑一次该部署脚本，确保线上是最新构建。

### B. IP 校验一直转圈或接口超时

已对 IP 校验做了超时和异常处理：超时或接口失败会直接放行，不再一直卡在“校验中”。若你本地代码已更新，需要重新构建前端并部署（见上面 A）。

### C. Nginx 配置不对

站点根目录应指向前端构建目录（例如 `root /var/www/younov/dist;`），且要有 SPA 回退：

```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

否则刷新子路径或直接访问会变成 404 或白屏。改完 Nginx 后执行 `nginx -t` 再 `systemctl reload nginx`。

---

## 3. 快速自检接口是否正常

在浏览器地址栏或本机 PowerShell 访问：

- `https://younov.com/api/ip/check`
- `https://younov.com/api/leaderboard`

若返回 JSON（哪怕是一段数据），说明 API 正常；若 404 或 502，说明后端或反代有问题，需先修 API/反代再看前端。

---

**总结**：先 F12 看 Console 和 Network 是否有 404 或报错；多数白屏是**前端 JS 没加载到**（404），需要在本机 `npm run build` 后把 `dist` 部署到服务器并确认 Nginx 指向正确。
